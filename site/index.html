    Deprecation vs Retirement What's the Difference?

    Deprecation marks a model version as no longer recommended. It still works, but no new deployments can be created. After deprecation you should begin migrating to the replacement model.

    Retirement means the model version is fully removed. API calls will return errors after the retirement date. Always migrate before the retirement date.

    Phase New Deployments Existing Deployments API Calls
    Active Allowed Running Working
    Deprecated Blocked Running Working
    Retired Blocked Removed Errors

    See full retirement details and Microsoft's model lifecycle docs.
